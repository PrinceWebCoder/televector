Televector API

August 6, 2021


Request


All queries to the Televector API must should be presented in this form: /api/%comma% with one of HTTP methods.

    If a id parameter is present, you can also use this form: /api/%comma%/%id%.

The total number of all queries in the Televector API is 13.

Send the following request to find out if the Televector API is working: /api, the working field equals true, if API is running.


1. Commas
  • get-movies
  • movies
  • get-cartoons
  • cartoons

2. Types
  • Movie
  • Cartoon
  • PartType
  • Delete

3. Roles
  • Producer
  • Spectator

4. Result fields
  • movies
  • movie
  • cartoons
  • cartoon
  • created
  • changes
  • delete

Available methods


We support GET, POST, PATCH and DELETE HTTP methods. The response contains a JSON object, which always has a Boolean field ok. If ok equals true, the request was successful, and the result of the query can be found in the any results field. In case of an unsuccessful request, ok equals false, and the error is explained in the error and its message is explained in the message (e.g. error: 404, message: Not found ). All queries must be made using UTF-8.

movies


Methods: GET, POST, PATCH, DELETE

1. Info part of movies:

Method: GET (public)

Use this to get information of part of movies. Returns an Array of PartType with the movies field on success.

URL: /api/movies

2. Create a new movie:

Method: POST
Roles: Producer

Use this to create a new movie. Returns an Movie with the created field on success.

Example:
Method: POST
JSON: Movie (all fields required)
URL: /api/movies

3. Edit a movie:

Method: PATCH
Roles: Producer

Use this to edit a movie. id required. Returns only you changed fields in Movie with the changes field on success.

Example:
Method: PATCH
JSON: Movie (fields optional)
URL: /api/movies/7

4. Delete a movie:

Method: DELETE
Roles: Producer

Use this to delete a movie. id required. Returns an Delete with the delete field on success.

Example:
Method: DELETE

URL: /api/movies/7

get-movies


Method: GET
Roles: Producer, Spectator

1. Get all movies:


Use this to get information of movies. Returns an Array of Movie with the movies field on success.

URL: /api/get-movies

2. Get movie with id:


Use this to get information of one movie. Returns an Movie with the movie field on success.

Example: /api/get-movies/7

Movie

This object represents a movie.
Field Type Description
id Integer ID of movie. Only used to delete movie, unnecessary for JSON body on request
title String Title of movie
year Integer Released year of movie
director String Director of movie
duration Integer Duration in minutes of movie
county String County that produced movie
image_link String Available image's link for movie


Cartoon

This object represents a cartoon.
Field Type Description
id Integer ID of cartoon. Only used to delete cartoon, unnecessary for JSON body on request
title String Title of cartoon
year Integer Released year of cartoon
director String Director of cartoon
duration Integer Duration in minutes of cartoon
county String County that produced cartoon
image_link String Available image's link for cartoon


PartType

This object represents a part of movies or part of cartoons.
Field Type Description
title String Title of movie or cartoon
year Integer Released year of movie or cartoon


Delete

This object represents ID of deleted resource.
Field Type Description
delete Integer ID of deleted movie or cartoon




Producer


[Admin]. This role can get full information of cartoons and movies, create new movies and cartoons, edit cartoons and movies and also delete them.

Spectator


[User]. This role only can get full information of movies and cartoons.
